# 知识管理平台首页设计说明

日期：2026-06-23

## 背景

当前项目是一个轻量静态站点，正式页面位于 `site/` 目录。现有版本以“平哥聊职教”的短视频精读为单一主题，数据来自 `site/data/articles.json`。

本次目标是把站点升级为现代化知识管理平台首页，首发两个板块：

- 产教融合：放入“平哥聊职教”
- 数字化转型：放入“数字周参”

用户已确认整体方向为“门户 + 工作台折中”：上半屏有现代网站格局和两个板块入口，下半屏快速进入搜索、筛选和内容阅读。

## 设计目标

1. 平台第一眼不再像单账号摘要页，而像一个可持续扩展的知识门户。
2. 首发只做两个板块，但导航、数据结构和内容卡片要预留未来增加板块、来源和专题的空间。
3. “平哥聊职教”和“数字周参”要作为来源归属呈现，而不是和板块概念混在一起。
4. 首页既能对外展示平台方向，也能满足高频检索和阅读。

## 信息架构

首页采用四层结构：

1. 顶部导航
   - 平台名称
   - 首页
   - 产教融合
   - 数字化转型
   - 来源
   - 检索

2. 首屏门户区
   - 平台主张：把行业观察整理成可检索、可复用的知识资产
   - 简短说明：围绕职业教育与数字化转型，沉淀来源、观点、案例、清单与精读笔记
   - 两个核心板块入口

3. 板块入口
   - 产教融合
     - 主要来源：平哥聊职教
     - 内容关键词：职业教育、校企合作、专业建设、产教融合项目、教师发展
   - 数字化转型
     - 主要来源：数字周参
     - 内容关键词：教育信息化、数字化转型、产业观察、政策趋势、案例分析

4. 内容工作台
   - 全站搜索
   - 板块筛选
   - 来源筛选
   - 标签筛选
   - 最新内容卡片
   - 文章或视频精读弹层/详情视图

## 页面结构

推荐使用单页静态应用结构：

- `site/index.html` 保留为主入口。
- `site/styles.css` 重写为现代门户风格。
- `site/app.js` 从单一账号渲染升级为平台内容渲染。
- `site/data/articles.json` 可调整为平台级数据，或新增 `site/data/content.json` 后让页面读取新数据。

首屏布局：

- 左侧或主区域：平台标题、定位语、平台统计。
- 右侧：两个板块入口卡片。
- 首屏底部露出内容工作台，让用户知道页面可以继续检索和阅读。

工作台布局：

- 左侧：板块与来源筛选。
- 右侧：搜索框、标签、内容卡片网格。
- 移动端：筛选区折叠到顶部，卡片单列展示。

## 数据模型

建议把每条内容抽象为统一内容项：

```json
{
  "id": "pg-001",
  "title": "学校宣传的“校企合作”家长怎么看真假",
  "source": "平哥聊职教",
  "sourceType": "douyin",
  "section": "产教融合",
  "publishedAt": "2026-06-23 18:04",
  "summary": "判断校企合作不能只看牌子、合影和协议...",
  "tags": ["校企合作", "家长择校", "产教融合"],
  "contentType": "video_digest",
  "body": "整理稿或精读正文",
  "note": "非逐字全文；根据可访问内容整理。",
  "url": "https://..."
}
```

“数字周参”内容可使用同一模型：

```json
{
  "source": "数字周参",
  "sourceType": "wechat",
  "section": "数字化转型",
  "contentType": "article_digest"
}
```

## 交互

1. 默认展示全部内容，按发布时间倒序排列。
2. 点击板块入口时，内容工作台切换到对应板块。
3. 搜索范围包括标题、摘要、标签、来源和正文。
4. 卡片显示标题、来源、板块、发布时间、摘要和标签。
5. 点击卡片打开详情阅读层，显示精读摘要、正文、备注和来源链接。
6. 空结果状态提示用户更换关键词或筛选条件。

## 视觉方向

整体采用现代知识门户风格：

- 克制、清晰、偏专业，不做营销落地页。
- 避免旧版单一复古纸张感，改为更现代的深浅对比、清晰分区和信息密度。
- 色彩建议使用多色但克制的系统：
  - 深墨色用于主视觉和导航。
  - 绿色系用于产教融合。
  - 蓝色系用于数字化转型。
  - 中性色用于背景、边线和阅读区。
- 卡片圆角不超过 8px。
- 首屏要有明确平台感，不能只显示单个账号名称。

## 首批内容归属

现有 `site/data/articles.json` 中“平哥聊职教”的 10 条内容全部归入“产教融合”板块。

“数字周参”归入“数字化转型”板块。首批内容可来自当前工作区已有文档和数据处理产物：

- `数字周参-最新文章摘录与总结.docx`
- `数字周参-近15天文章检索处理报告.docx`
- `数字周参-教育信息化行业详细精读版.docx`
- `output/数字周参最新三篇/数字周参最新三篇文章检索汇总.docx`

实现时需要先检查是否已有可结构化 JSON 数据；若没有，先从已有报告或文档中整理最小真实条目接入，不使用虚构内容。

## 非目标

本阶段不做：

- 登录系统
- 后台管理
- IMA 入库流程
- 多用户权限
- 复杂知识图谱
- 自动抓取微信公众号或抖音内容

这些能力可以后续作为独立迭代设计。

## 验收标准

1. 首页标题、导航和视觉重心体现“知识管理平台”，不再是单一“平哥聊职教”页面。
2. 页面明确展示两个板块：产教融合、数字化转型。
3. “平哥聊职教”内容进入产教融合板块。
4. “数字周参”内容进入数字化转型板块。
5. 用户可以搜索、按板块筛选、按来源筛选。
6. 点击内容卡片可以查看详情。
7. 桌面和移动端都无明显文字溢出或布局重叠。
