# Project Agent Rules

## 微信公众号处理流程

当用户要求处理微信公众号文章、公众号历史文章、公众号最新/上一篇文章，并要求总结、形成 Word、放入 IMA 知识库时，按以下流程执行。

### 1. 定位文章

- 优先使用用户已登录微信，通过公众号会话、搜一搜、公众号主页或历史消息定位文章。
- 如果微信界面不稳定或无法直接取到链接，可用公开转载页、搜索引擎、站内搜索、已知原文链接和网页元数据交叉核验。
- 涉及“最新一篇”“上一篇”“按时间顺序往前一篇”时，必须记录并核对文章发布时间。若无法完全确认时间顺序，要在结果中说明“当前可核验的上一篇/候选文章”，不要臆造。
- 保存文章元信息：标题、公众号、作者、发布时间、原文链接、核验来源链接。

### 1.1 发布时间抽取规则

- 微信公众号文章入库、建站或追加到知识管理平台时，必须为每篇文章尽量补齐发布时间。
- 发布时间优先从公众号原文页公开 HTML 中抽取；对 `mp.weixin.qq.com` 文章，可用 `curl --http1.1 --tlsv1.2 -L` 搭配常见浏览器或微信 UA 获取页面，再解析 `ori_create_time`、`oriCreateTime`、`publish_time`、`var ct` 等字段。
- 若解析到 10 位时间戳，按本地时区转换为：
  - `publishedAt: "YYYY-MM-DD HH:mm"`
  - `publishedLabel: "YYYY-MM-DD"`
- 如果 IMA `get_knowledge_list` 或 `get_media_info` 返回发布时间字段，也可优先使用接口字段；但不得把 IMA 导入时间、抓取时间、文件生成时间冒充为文章发布时间。
- 如果确实无法核验发布时间，数据中可留空或保留核验说明；前端卡片不要显示“待核验时间”“待核验”等占位文本。

### 2. 版权与内容处理

- 不要整篇逐字复制受版权保护的公众号文章到最终文档，除非用户提供全文并确认有转载/使用授权。
- 默认输出“详细精读版”：按原文结构、观点和论证路径进行改写、归纳、扩展说明，尽量覆盖主要信息，但不用原文逐字复刻。
- 可以保留少量必要短摘录用于识别主旨，但总量应很少，并明确标注“非全文转载”。

### 3. Word 文档生成

- 使用 documents 技能与工作区依赖生成 `.docx`。
- 默认文档形态为“详细精读版”，建议结构：
  - 文章基本信息
  - 全文主旨
  - 开篇问题意识/背景
  - 原文核心论证按小节展开
  - 关键概念解释
  - 对不同角色/场景的启示
  - 行动清单或转型清单
  - 精读结论
  - 来源链接
- 文档命名建议：`公众号名-文章标题-详细精读版.docx`。文件名要避免特殊符号导致上传失败。
- 生成后必须用 `render_docx.py` 渲染页面 PNG 并抽查首页、中间页、末页；如渲染失败，需说明原因并至少做结构检查。

### 4. IMA 知识库入库

- 当用户说“放到 IMA/ima/知识库/大明的知识库”时，必须使用 `ima-skill`。
- 先确认凭证：`~/.config/ima/client_id` 和 `~/.config/ima/api_key`，或环境变量 `IMA_OPENAPI_CLIENTID`、`IMA_OPENAPI_APIKEY`。
- 用户指定“大明的知识库”时，用 `search_knowledge_base` 搜索知识库名称，获取 `kb_id`，不要猜 ID。
- 上传 Word 文件到知识库时，按 knowledge-base 文件上传安全门执行：
  - `preflight-check.cjs`
  - `check_repeated_names`
  - 若重名，优先采用追加时间戳的新文件名保留两者，除非用户明确取消
  - `create_media`
  - `cos-upload.cjs`
  - `add_knowledge`
- 文件上传时 `title` 必须等于最终上传文件名（含扩展名）。
- 如只是保存原文链接，可额外调用 `import_urls`，但不要用链接入库替代用户要求的精读 Word 文档，除非用户明确只要链接。

### 5. 最终回复

- 简要说明处理了哪篇文章、发布时间、文档路径、是否已上传到哪个知识库。
- 如无法确认“上一篇”的严格顺序，要明确说明核验边界。
- 不要输出渲染中间图片、临时文件或 API 凭证信息。

## 抖音短视频转写与网站整理流程

当用户要求处理抖音账号、抖音视频、最新/最近若干条短视频，并要求“语音转文字”“整理原文”“总结”“做成网站”时，按以下流程执行。

### 1. 定位视频

- 优先使用用户本机已登录的抖音客户端或浏览器抖音页面，通过账号搜索、账号主页、视频列表定位目标账号。
- 记录账号元信息：账号名称、抖音号、主页链接或可核验页面。
- 按用户要求的数量采集视频；若用户要求“最新”“最近”“按日期由近到远”，必须按发布时间排序，新视频靠前。
- 对每条视频保存元信息：标题/文案、发布时间或相对发布时间、时长、作者、视频链接或账号主页链接、核验来源。
- 若只能从页面读取相对时间或主页顺序，必须标注“页面核验时间”和“按主页顺序推定”，不要把推定日期说成已核验的绝对发布日期。

### 2. 转写优先级

- 优先获取平台可见字幕、视频详情页字幕字段、公开字幕文件或用户提供的音频/视频文件。
- 若能合法取得音频文件，并且本机有可用转写工具，可进行语音识别转写；转写后应做中文断句、去口癖、修正明显识别错误。
- 若无法取得音频、页面没有暴露字幕，或本机缺少转写工具，不要伪造逐字稿；输出应命名为“语音整理稿”“基于可见字幕/标题的整理稿”或“非逐字稿”。
- 对“原文”栏目，除非确实完成逐字转写或用户提供全文授权，否则不得标为“完整原文”或“逐字全文”。
- 如存在自动转写不确定处，应保留“不确定/待核验”说明，或用概括性整理替代不可靠逐字内容。

### 3. 版权与内容边界

- 不要未经授权整段复制或批量复刻第三方短视频完整口播全文到最终公开页面。
- 默认输出“总结 + 语音整理稿”：覆盖主要观点、论证路径和行动建议，但以改写、归纳、结构化整理为主。
- 可保留少量必要短句用于识别主题，但必须避免把短视频内容完整逐字搬运成可替代原视频的文本。
- 页面或文档中要清楚标注数据来源、转写方式和核验边界，例如“非逐字全文”“根据可见字幕与标题整理”。

### 4. 网站生成

- 使用本地静态网站优先：`site/index.html`、`site/styles.css`、`site/app.js`、`site/data/*.json`。
- 网站默认结构：
  - 账号信息与采集时间
  - 检索框与主题筛选
  - 视频卡片列表，按发布时间由近到远排序
  - 每条视频展示：标题、发布时间/相对时间、时长、摘要、主题标签
  - 详情页或弹窗展示：总结、语音整理稿、核验/版权说明、来源链接
- 若日期不能完全核验，卡片和详情中都要保留“按主页顺序推定”或等价说明。
- 生成后启动本地静态服务或提供可直接打开的 HTML 路径，并做基本校验：JSON 可解析、首页可访问、列表数量正确、排序正确。

### 5. 最终回复

- 简要说明处理的账号、整理的视频数量、排序依据、网站路径或本地预览地址。
- 明确说明转写方式：逐字转写、自动字幕转写、或基于可见字幕/标题的整理稿。
- 如未能取得完整音频或字幕，要直说限制，不要声称已完成完整语音转文字。
- 不要输出第三方视频的完整逐字稿、临时缓存路径或与用户账号相关的敏感信息。

## 知识管理平台汇聚规则

每次完成文章、视频、报告、网站、Word 文档、数据表或其他知识整理任务后，必须把结果汇聚到当前项目“知识管理平台”中，作为默认收尾动作执行。

### 0. 模块归类

- “平哥聊职教”的所有内容默认归入“产教融合”模块。
- 写入网站或结构化数据时，必须设置 `section: "产教融合"`，并设置 `module: "产教融合"`；如有 `moduleId`，使用 `industry-education`。
- 在知识管理平台索引、资源清单、README 和网站卡片中，都要明确显示其所属模块为“产教融合”。
- 后续继续追加“平哥聊职教”视频、转写、摘要或资源时，必须追加到“产教融合”模块下，不要新建重复模块。
- “平哥聊职教”新增内容不能只写入专题数据或归档文件；还必须同步到网站主页数据源 `site/data/content.json`，确保知识管理平台首页、筛选器和卡片列表都能看到新增资源。
- “数字周参”的所有微信公众号文章默认归入“数字化转型”模块。
- 写入网站或结构化数据时，必须设置 `section: "数字化转型"`，并设置 `module: "数字化转型"`；如有 `moduleId`，使用 `digital-transformation`。
- “数字周参”新增内容不能只写入 `site/data/articles.json` 或 `site/data/ima_raw.json`；还必须同步到主页聚合数据源 `site/data/content.json`，确保知识管理平台首页、筛选器和卡片列表都能看到新增资源。

### 1. 汇聚范围

- 需要汇聚的产物包括：`.docx`、`.pdf`、`.md`、`.html`、`.json`、`.csv`、网站目录、渲染后的正式成果、整理后的原始数据、摘要和核验记录。
- 临时文件、缓存文件、渲染中间图、API 凭证、浏览器缓存、未脱敏日志不要汇聚。
- 如果用户明确要求上传到 IMA/知识库，还要按对应 IMA 流程执行；本地知识管理平台汇聚不能替代 IMA 入库。

### 2. 本地归档

- 默认在项目内使用 `docs/` 或 `output/` 存放正式成果；网站类成果继续使用 `site/`。
- 每次任务建议建立清晰的主题目录，例如：`output/抖音-平哥聊职教-短视频精读/`、`output/公众号名-文章标题/`。
- 归档时保留：
  - 最终成果文件
  - 结构化数据文件
  - 来源与核验说明
  - 生成时间
  - 已知限制与待核验事项
- 文件名避免特殊符号，尽量使用中文标题 + 类型后缀，如 `平哥聊职教-最近30条短视频精读.json`、`公众号名-文章标题-详细精读版.docx`。

### 3. 平台索引

- 每次汇聚后，更新或创建一个索引文件，优先使用 `docs/knowledge-index.md`。
- 索引记录至少包含：
  - 任务名称
  - 处理对象
  - 产物类型
  - 产物路径
  - 来源链接或核验来源
  - 处理时间
  - 版权/转写/核验边界
  - 是否已上传 IMA，以及目标知识库名称
- 若任务生成网站，应在索引中记录本地预览地址或可打开的 `index.html` 路径。

### 4. 网站数据同步

- 如果任务产物适合浏览，应同步进入 `site/data/` 的结构化数据，并确保 `site/index.html` 或对应页面能访问。
- 更新网站后必须做基本校验：数据可解析、数量正确、排序正确、页面可打开。
- 对已有专题数据文件（如 `site/data/articles.json`）继续追加时，要同步检查主页聚合数据（如 `site/data/content.json`）是否包含同样数量的正式条目；若数量不一致，必须补齐后再回复用户。
- 如果新任务与已有网站主题不同，可新增数据文件或页面，不要随意覆盖无关主题数据；覆盖前必须确认这是用户想要的聚合方式。
- 网站首页卡片的来源与发布时间应放在同一行展示，例如：`数字周参 · 发布时间：2026-06-21` 或 `平哥聊职教 · 发布时间：2026-06-23 · 01:59`。
- 对“数字周参”等微信公众号内容，卡片必须优先显示真实文章发布时间；若无法核验，卡片只显示来源，不显示“待核验时间”“待核验”等占位文本。
- 更新“数字周参”网站数据后必须校验：`site/data/content.json`、`site/data/articles.json`、`site/data/ima_raw.json` 中的数字周参数量一致；所有已核验文章都有 `publishedAt` 和 `publishedLabel`；`site/app.js` 语法检查通过；本地服务能读到最新 JSON。

### 4.1 网站启动命令

当用户要求“启动网站”“打开知识管理平台”“预览网站”或类似请求时，默认在当前项目执行以下命令：

```bash
cd /Users/david/Documents/开发代码/知识管理平台/site
python3 -m http.server 8787
```

- 默认预览地址：`http://127.0.0.1:8787/`
- 如果 `8787` 端口已被占用，改用相邻空闲端口，例如：

```bash
cd /Users/david/Documents/开发代码/知识管理平台/site
python3 -m http.server 8788
```

- 启动后必须用 `curl -fsS http://127.0.0.1:8787/` 或对应端口做首页可访问校验；涉及数据更新时，还必须校验 `site/data/content.json` 可解析且数量正确。

### 5. 最终回复

- 最终回复除了说明任务结果，还必须说明已汇聚到知识管理平台的路径。
- 如果因信息不完整、工具限制或用户要求暂不归档，必须明确说明未汇聚的原因和后续可执行动作。
